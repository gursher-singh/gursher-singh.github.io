Scan-to-Desktop Service Deployment

Ontario Public Service  ·  May – Aug 2025

Tasked with validating and advancing a Scan-to-Folder deployment for Windows 11 endpoints across provincial park locations. Park sites operate off the corporate network, which caused device policies to treat local connections differently than expected - exposing gaps that only surfaced in the field. Diagnosed each issue, implemented approved remediations, and reported findings that shaped how the process was refined before broader rollout.

DIAG

Diagnosing Silent Scan Failures with Wireshark

Scans were failing at certain locations with no useful error from the printer or Windows. Park sites operate off the corporate network, so device policies classify the local network differently - behaving more like a home network than a managed corporate network. Packet analysis was the only reliable way to determine what was actually being blocked and where.

WiresharkSMB2/SMB3Packet CaptureTCP/IP
Problem
Because these endpoints sit off the corporate network, standard policy assumptions didn't hold. The local network was being treated as untrusted by the OS, which changed how firewall profiles, share access, and SMB negotiation behaved - none of which was obvious from the surface when a scan failed silently.
Approach
Captured live SMB traffic between the Ricoh MFP and the Windows 11 endpoint during a scan attempt. Filtered for SMB session setup to locate exactly where the connection was failing. Each failure pattern was distinguishable - a reset response indicated a firewall block, an access-denied response indicated a permissions issue, and no response at all indicated a network path problem. This gave a clear, repeatable method for categorizing failures at each site.
Outcome
Each site's failure was identified and categorized with precision - providing the team with specific, reproducible findings rather than general reports. This shifted the remediation effort from reactive trial-and-error to targeted, evidence-based fixes.
FW

Firewall Profile Misclassification on Off-Network Endpoints

The most common issue found: Windows 11 was classifying the local park network under the public firewall profile rather than private, which restricted inbound traffic in ways the deployment process hadn't accounted for. This was a field-only finding - not visible from the design stage.

Windows Defender FirewallInbound RulesNetwork Profile ClassificationOff-Network Endpoints
Problem
Park endpoints are domain-joined, but because they operate off the corporate network, inbound SMB traffic from non-corporate sources was being blocked by policy. The deployment process assumed internal network conditions - it didn't account for sites where the scanner and the endpoint were on a network the domain treated as external.
Approach
Confirmed via packet capture that SMB traffic from the scanner was being dropped at the firewall level. Proposed a targeted remediation - a scoped inbound firewall rule allowing scanner traffic specifically, without loosening the broader policy posture - and got it approved before implementing. Validated the fix end-to-end at the affected sites. Findings were reported to the senior team, and the approved rule configuration was later incorporated into the deployment policy applied at scale.
Outcome
Remediation validated and confirmed working at pilot sites. The firewall gap - invisible during process design - was documented and fed back to the team in time to be addressed in policy before the province-wide rollout, preventing it from recurring at every subsequent location.
PERM

Permission Configuration & Confidentiality Controls

Permission setup varied significantly across locations - partly because sites had different device types. Shared workstations used by multiple staff required a different access model than single-user devices, and printer settings needed adjustments to maintain confidentiality in a government environment.

SMB SharesNTFS PermissionsRicoh MFP Admin PortalConfidentiality Controls
Problem
The deployment process used a single permission model for all endpoints, but some sites had a mix of shared workstations and single-user devices. A shared device needed folder permissions scoped so each user's scans landed in their own designated location - not a single shared destination accessible to everyone. Additionally, default Ricoh MFP settings allowed scanned documents to be visible in the printer's job history, which was a confidentiality concern for government documents.
Approach
Configured custom SMB share and NTFS permission sets depending on device type - shared workstations received user-scoped folder structures with per-user write access, while single-user devices used simpler single-destination configurations. On the printer side, modified Ricoh MFP settings via the admin portal to allow user login on the printer panel, ensuring scanned content wasn't accessible to subsequent users.
Outcome
Permission configurations tailored to actual site conditions rather than a one-size-fits-all model. Confidentiality controls on the MFP ensured scanned government documents weren't exposed.
REPORT

Identifying Gaps in the Deployment Documentation

The provided deployment guide was missing steps that only became apparent during field testing. Flagging these gaps early meant the team could update the process before it was handed to technicians who would follow it at scale.

Field TestingProcess DocumentationGap AnalysisReporting
Problem
The deployment document had been written based on expected behaviour in a standard corporate environment. Field testing revealed things like local account password expiry. Windows defaults this to 30 days, which would silently break scanner authentication without any alert. The guide also didn't account the different permission models required for shared versus single-user devices.
Approach
Logged each gap as it was encountered - what the document said, what actually happened in the field, and what step was missing. Compiled findings into a structured report covering the local account expiry issue, permission model discrepancies, and printer confidentiality settings. Submitted the report to the senior team with suggested additions to the deployment checklist for each gap identified.
Outcome
The team received a field-tested analysis of the deployment documentation before the process went to scale. Issues like the local account password expiry and different permissions model for shared devices, were caught and addressed in the updated process rather than discovered in production.
